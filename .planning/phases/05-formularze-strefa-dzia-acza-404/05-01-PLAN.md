<task type="auto">
  <name>Task 2: Create auth.ts + middleware.ts + NextAuth route handler</name>
  <files>auth.ts, middleware.ts, app/api/auth/[...nextauth]/route.ts</files>
  <action>
    Create three files implementing NextAuth v5 pattern (Auth.js v5 — uses AUTH_ env var prefix, root-level auth.ts):

    **auth.ts** (root level, NOT inside app/):
    ```typescript
    import NextAuth from 'next-auth'
    import Google from 'next-auth/providers/google'

    export const { handlers, auth, signIn, signOut } = NextAuth({
      providers: [
        Google({
          authorization: {
            params: {
              hd: 'samorzad.ue.wroc.pl', // UI hint to Google — shows domain accounts first
            },
          },
        }),
      ],
      session: { strategy: 'jwt' },
      callbacks: {
        async signIn({ profile }) {
          // Server-side enforcement — hd param alone is NOT security (it's just a hint)
          return profile?.email?.endsWith('@samorzad.ue.wroc.pl') ?? false
        },
        async jwt({ token, profile }) {
          if (profile) {
            token.email = profile.email
          }
          return token
        },
      },
    })
    ```

    **middleware.ts** (root level — Edge Runtime compatible, JWT only, no DB):
    ```typescript
    import { auth } from './auth'

    export default auth((req) => {
      const isLoggedIn = !!req.auth
      const isProtected = req.nextUrl.pathname.startsWith('/strefa-dzialacza')

      if (isProtected && !isLoggedIn) {
        // Redirect to strefa-dzialacza landing (which shows login UI)
        const loginUrl = new URL('/strefa-dzialacza', req.url)
        return Response.redirect(loginUrl)
      }
    })

    export const config = {
      matcher: ['/strefa-dzialacza/:path+'],
    }
    ```

    Note: matcher uses `:path+` (one or more segments) so the landing page `/strefa-dzialacza` itself is NOT protected by middleware — it handles its own conditional render. Sub-routes like `/strefa-dzialacza/dashboard` and `/strefa-dzialacza/lista-obecnosci` ARE protected.

    **app/api/auth/[...nextauth]/route.ts** (thin wrapper — all logic is in auth.ts):
    ```typescript
    import { handlers } from '@/auth'
    export const { GET, POST } = handlers
    ```

    Env vars required (do NOT create .env.local — user_setup documents what to add):
    - AUTH_SECRET — required, NextAuth refuses to start without it
    - AUTH_GOOGLE_ID — required for Google provider
    - AUTH_GOOGLE_SECRET — required for Google provider
