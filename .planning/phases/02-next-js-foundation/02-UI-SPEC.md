## Spacing Scale

8-point base scale. All values are multiples of 4px. Tailwind's default scale is used directly —
no custom spacing extension needed.

| Token | px value | Tailwind class | Primary usage |
|---|---|---|---|
| xs | 4px | `p-1`, `gap-1` | Icon internal gaps |
| sm | 8px | `p-2`, `gap-2` | Compact badge padding, tag gaps |
| md | 16px | `p-4`, `gap-4` | Default element spacing, card gap |
| lg | 24px | `p-6`, `gap-6` | Card padding, section header gap |
| xl | 32px | `p-8`, `gap-8` | Section inner padding |
| 2xl | 48px | `p-12`, `gap-12` | Footer column gaps |
| 3xl | 64px | `p-16` | Footer top/bottom padding |

Section padding: `py-20` (80px) on desktop, `py-14` (56px) on mobile (`max-768px`).
This ports Phase 1's `section { padding: 5rem 0 }` (80px) and mobile `section { padding: 3.5rem 0 }` (56px).

Container: `max-w-brand mx-auto px-6` — matches Phase 1 `.container { max-width: 1200px; padding: 0 1.5rem }`.

Exceptions:
- Navbar height: `h-[70px]` — Phase 1 mobile-menu top offset is `top: 70px`
- Hero section: `pt-20 min-h-screen` — accommodates fixed navbar (80px top padding in Phase 1)
- Subpage hero: `pt-36 pb-16` — matches Phase 1 `.page-hero { padding: 9rem 0 4rem }`
- Social icon touch targets: `w-[38px] h-[38px]` — Phase 1 `.social-link` exact dimensions
- Avatar image: `w-[100px] h-[100px]` — Phase 1 `.avatar-img` exact dimensions
- Zone icon box: `w-16 h-16` (64px) — Phase 1 `.zone-icon`

60 / 30 / 10 split:

| Role | Hex | % | Usage |
|---|---|---|---|
| Dominant surface | `#FFFFFF` | 60% | Page background, card backgrounds, navbar (post-scroll) |
| Secondary surface | `#F0F7FF` | 30% | Alternate section backgrounds (`section-alt`), input backgrounds, stat cards |
| Accent | `#3BAEFF` | 10% | See reserved list below |
| Accent dark | `#1A8FE3` | — | Button hover, link hover states only |

- Primary CTA button fill
- Active navbar link text and underline
- Badge background (as `#EAF5FF`) + text
- Section label pill backgrounds (as `#EAF5FF`) + text
- Card hover border
- Social icon hover fill (background)
- Logo icon box background
- Counter / stat numbers

- Footer email link text

It is NOT used for: body text, card borders (normal state), section backgrounds.

Footer background: `ssuew-black` (`#1A2340`) — dark surface, white text.

Navbar states:
- Before scroll: `bg-transparent` — overlays hero gradient
- After scroll (20px threshold): `backdrop-blur-md bg-white/80 shadow-sm` — glassmorphism

## Navbar — Detailed Interaction Contract

### Structure

```
[Logo: logo.png 40×40px] [SSUEW wordmark]          [nav links ...]   [Strefa Działacza btn]
```

### Desktop

- Height: `h-[70px]` fixed, `z-50`
- Logo: `<Image>` src `/assets/logos/logo.png`, size `40×40`, alt "Logo Samorządu Studentów UEW"
- Wordmark: "Samorząd UEW" — font-sans weight-800 text-base text-ssuew-black
- Nav links: font-sans 0.9rem weight-500 text-ssuew-black, hover → text-primary, transition-colors duration-300
- Active link: text-primary weight-600 (detected via `usePathname()`)
- Dropdown trigger: parent `<div class="relative group">` — no click required
- Dropdown panel: `absolute top-full left-0`, appears on `group-hover`, bg-white shadow-brand rounded-brand-sm min-w-[192px] py-2
- Dropdown items: `block px-4 py-2 text-sm text-ssuew-black hover:text-primary hover:bg-ssuew-gray-100`
- Dropdown animation: CSS `opacity-0 group-hover:opacity-100 translate-y-1 group-hover:translate-y-0 transition-all duration-200 pointer-events-none group-hover:pointer-events-auto`
- "Strefa Działacza" button: pill shape `rounded-full bg-primary text-white px-5 py-2 text-sm font-semibold hover:bg-primary-dark transition-colors duration-300`
- Scroll threshold: `window.scrollY > 20` → add glassmorphism class

### Glassmorphism state (scrolled)

```
backdrop-blur-md bg-white/80 shadow-[0_2px_20px_rgba(0,0,0,0.08)]
```

Before scroll: `bg-transparent` (no backdrop). Transition: `transition-all duration-300`.

### Mobile (max-768px)

- Hamburger button: `w-10 h-10 flex flex-col gap-[5px] items-center justify-center p-2`
- Three bars: `block w-6 h-[2px] bg-ssuew-black rounded-sm transition-all duration-300`
- Open state: top bar rotates 45deg, middle bar opacity-0, bottom bar rotates -45deg (X shape)
- Mobile drawer: `position: fixed, top: 70px, left: 0, right: 0` — slides down from top
- Drawer animation: Framer Motion `AnimatePresence` + `motion.div initial={{ height: 0, opacity: 0 }} animate={{ height: 'auto', opacity: 1 }} exit={{ height: 0, opacity: 0 }} transition={{ duration: 0.3, ease: 'easeInOut' }}`
- Drawer background: `bg-white border-b border-ssuew-gray-200 shadow-[0_8px_30px_rgba(0,0,0,0.12)]`
- Drawer padding: `px-6 py-6 flex flex-col gap-4`
- Top-level links without children: `text-base font-medium text-ssuew-black py-2 border-b border-ssuew-gray-200`
- Top-level links with children (accordion): chevron icon (lucide `ChevronDown`), rotates 180deg when open
- Submenu items: `pl-4 text-sm text-ssuew-gray-600 py-1.5`
- Submenu accordion: Framer Motion height animation, same pattern as drawer
- "Strefa Działacza" in mobile drawer: `block w-full text-center rounded-full bg-primary text-white px-5 py-3 font-semibold mt-2`
- Body scroll lock when drawer open: `document.body.style.overflow = 'hidden'` / `'auto'`
- Drawer closes on: route change (usePathname effect), Escape key, overlay click

## Scroll Animations — Framer Motion Contract

### FadeUp component (`components/ui/FadeUp.tsx`)

This is the ONLY scroll animation component needed for Phase 2.

| Property | Value | Source |
|---|---|---|
| Initial state | `opacity: 0, y: 30` | Phase 1 `.fade-up { opacity: 0; transform: translateY(30px) }` |
| Animated state | `opacity: 1, y: 0` | Phase 1 `.fade-up.visible { opacity: 1; transform: none }` |
| Duration | `0.6s` | Phase 1 `transition: opacity 0.6s ease, transform 0.6s ease` |
| Easing | `easeOut` | Phase 1 `ease` |
| Viewport margin | `-80px` | Triggers slightly before element fully visible |
| Repeat | `once: true` | Phase 1 — no re-trigger on scroll-back |
| Reduced motion | Skip animation entirely, render `<div>` passthrough | ANIM-02 requirement |

Stagger delay values — exact port of Phase 1 `.delay-N`:

| Delay prop | Value | Phase 1 equivalent |
|---|---|---|

| `delay={0.3}` | 300ms | `.delay-3` |
| `delay={0.4}` | 400ms | `.delay-4` |

Usage in Phase 2: Wrap Navbar placeholder section and Footer columns in `<FadeUp>` for demonstration. All Phase 3+ sections will use FadeUp.

## Lenis Smooth Scroll Contract

### Configuration

```ts
options: {
  duration: 1.2,      // scroll physics duration in seconds
  syncTouch: false,   // don't override native touch scroll on iOS (preserves momentum)
  easing: (t) => Math.min(1, 1.001 - Math.pow(2, -10 * t)),  // default exponential
}
```

`duration: 1.2` gives a premium feel without being sluggish. Phase 1 had no smooth scroll.

### Integration pattern

`<ReactLenis root options={...}>` from `lenis/react` wraps entire app in `LenisProvider`.
`lenis/dist/lenis.css` imported in `app/layout.tsx` (not in provider).

### Conflict prevention

Remove any `scroll-behavior: smooth` from `globals.css`. Lenis handles all scroll physics.
This prevents the Pitfall 4 identified in RESEARCH.md.
